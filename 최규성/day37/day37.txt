DI 자동주입

 XML 파일이 너무 커지는 것을 방지
 자동 주입 기능 사용시 스프링이 알아서 의존 객체를 찾아서 주입
 자동 주입 기능 사용방법
 - xml파일에  <context:annotation-config /> 설정을 추가
 - 자바파일에 의존주입대상에 @Autowired 또는 @Resource 설정
 - 스프링 MVC에서는 @Injection을 쓸 수도 있다.


@Autowired

java설정
-변수 / 생성자 / set메소드 설정

변수, 메소드가 여러개인 경우,
우선적으로 class내의 타입이 맞는 변수,메소드중에 제일 먼저 선언된 것을 우선적으로 출력하며
class의 타입이 같은 경우는 id를 보고,
또 class에 하나밖에 없는 경우는 id속성을 태그에서 생략해도 된다



객체찾기 우선 순위
1.type matching
2.type으로 찾은 객체가  여러개일경우 id 매칭

 -타입이 같은  빈을 검색하여 한 개면 그 빈 객체를 사용
 -두개이상이면 @Qualifier가 명시되어있는 빈 객체를 찾는다 명시된 값과 같은  빈 객체를 사용
 -두개이상이고 @Qualifier가 없을경우 이름이 같은 빈 객체를 찾아서 사용
 -위의  경우에 해당하는 객체가 없을 경우 예외가 발생함

component-scan

xml파일설정을 통해서 자동으로 빈으로 사용 될 객체를 등록한다

- <context:component-scan base-package="kr.co.mlec" />
- 지정된 패키지(kr.co.mlec) 하위의 모든 패키지를 스캔하여 빈으로 등록
- 빈으로 등록되려면 자바클래스에서 어노테이션을 사용해야한다
- @Component, @Controller, @Service, @Repository
- 컴포넌트 스캔으로 만들어진 빈객체의 id는 클래스의 첫글자 대문자를 소문자로 적용된다
